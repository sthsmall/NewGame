# 风险登记表

## 目的

在技术验证期间跟踪已知风险，使项目在核心不确定性解决之前不会漂移到生产阶段。

## 视觉风险

### HD-2D 风格可能难以阅读

风险：

- 像素角色可能看起来与3D环境脱节。
- 相机角度可能使移动或战斗不够清晰。

缓解措施：

- 尽早使用占位资源进行验证。
- 在最终美术之前调整比例、阴影、光照和相机。
- 在相同的相机设置下测试移动和战斗。

### Sprite3D 朝向可能看起来不正确

风险：

- 广告牌精灵可能感觉扁平或旋转异常。
- 方向动画可能与移动不匹配。

缓解措施：

- 测试不同的广告牌模式。
- 测试4方向或8方向的精灵集。
- 初始时保持玩家移动和朝向规则简单。

### 后处理可能隐藏游戏内容

风险：

- 雾效、辉光和深度效果可能使战斗更难阅读。

缓解措施：

- 将后处理视为可调节的。
- 以战斗可读性为优先。
- 在核心循环正常工作之前保持效果微妙。

## 战斗风险

### 攻击范围可能不清晰

风险：

- 倾斜相机可能使判断命中距离变得困难。

缓解措施：

- 使用可见的攻击效果。
- 早期让命中判定框范围宽裕。
- 开发期间添加调试可视化。

### 碰撞和精灵比例可能不一致

风险：

- 玩家可能视觉上命中或未命中，但与碰撞不匹配。

缓解措施：

- 保持碰撞简单。
- 使用一致的比例规则。
- 在添加精细美术之前使用调试形状进行验证。

### 敌人AI可能扩展过快

风险：

- AI复杂性可能在核心战斗循环被验证之前消耗时间。

缓解措施：

- 从一个敌人和一种攻击开始。
- 使用小型状态机。
- 除非场景需要，否则推迟路径寻找。

## 资源风险

### 资源风格可能冲突

风险：

- 社区资源和AI美术可能不具有一致的风格。

缓解措施：

- 在早期验证期间接受混合占位符。
- 使用颜色分级和光照来统一场景。
- 仅在系统验证之后才选择最终的视觉方向。

### 许可可能变得不明确

风险：

- 如果来源未追踪，资源可能无法用于商业用途。

缓解措施：

- 在资源登记表中记录每个外部资源。
- 优先选择 CC0 或明确商业友好的资源。
- 避免没有明确许可条款的资源。

### AI资源可能需要大量清理

风险：

- AI生成的精灵可能无法直接用于生产。

缓解措施：

- AI主要用于概念艺术和情绪探索。
- 手动清理或重绘选择用于生产的资源。
- 在测试之前不要依赖AI生成一致的动画。

## 范围风险

### 故事可能在系统验证之前扩展

风险：

- 写作和内容制作可能分散技术验证的注意力。

缓解措施：

- 让首个演示的故事内容保持精简。
- 使用临时的交互文本。
- 优先验证视觉、交互和战斗。

### 过多的RPG系统过早引入

风险：

- 背包、任务、存档/读档和游戏进程可能拖慢验证速度。

缓解措施：

- 推迟这些系统。
- 仅构建技术切片所需的部分。

## 当前最高风险

1. 在 HD-2D 相机下的实时战斗可读性
2. 像素角色与3D场景的视觉连贯性
3. 资源风格和许可管理
4. 核心循环验证前的范围膨胀
