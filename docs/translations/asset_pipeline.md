# 资源管线策略

## 目标

项目应避免将早期开发时间花在最终美术上。首个技术演示应使用以下实际组合：

- 社区资源
- AI辅助美术
- 简单的手工占位符
- Godot原生布局几何体

目的是在投入最终生产资源之前验证视觉方向和游戏系统。

## 资源来源优先级

技术验证期间的优先顺序：

1. 现有的 Godot 兼容工具和插件
2. CC0 或宽松许可的社区资源
3. 具有明确商业条款的付费资源包
4. AI生成的概念或占位美术
5. 手工制作的项目专用资源

避免使用任何没有明确许可的资源。

## 社区资源来源

值得研究的来源：

- Godot 资源库
- Godot 资源商店
- itch.io 游戏资源包
- OpenGameArt
- Kenney 资源
- Godot 插件和示例的 GitHub 仓库

典型用途：

- 角色占位符
- 敌人占位符
- UI图标
- 粒子和特效
- 音效
- 低多边形道具
- 瓦片纹理
- 测试环境

## AI辅助美术用途

AI美术适用于：

- 角色概念图
- 场景情绪板
- 配色方案探索
- 敌人概念图
- 道具概念图
- UI风格参考
- 单帧占位精灵
- 特效纹理实验

AI美术在以下方面不太可靠：

- 一致的动画帧
- 可用于生产的像素行走循环
- 多角度角色一致性
- 无需后处理的干净透明精灵
- 风格统一的资源包

推荐方法：

1. 使用 AI 进行概念和情绪探索。
2. 选择一到两个有希望的视觉方向。
3. 将关键资源转换或重绘为干净的像素精灵。
4. 使用 Aseprite、LibreSprite、Krita 或类似工具进行清理。
5. 在资源登记表中记录来源、工具、提示词、日期和用途。

## HD-2D 资源方向

HD-2D 外观应由以下元素组合构建：

- 3D布局几何体
- 像素角色
- 像素或低分辨率纹理
- 正交倾斜相机
- 光照
- 雾效
- 辉光
- 深度和大气效果
- 精细的比例和色彩控制

项目不需要完整的 HD-2D 资源包即可开始。早期验证可以使用简单的 3D 网格和占位像素精灵。

## 导入指南

像素精灵：

- 在适当时禁用纹理过滤。
- 保持精灵比例一致。
- 优选清晰的轮廓。
- 尽早测试 Sprite3D 和 AnimatedSprite3D。
- 验证精灵在相机下保持清晰。

3D环境：

- 先使用简单网格。
- 保持碰撞简单。
- 首次验证期间避免过于详细的资源。
- 优选可重复使用的模块化道具。
- 使用光照和雾效营造氛围，而非仅依赖几何体。

特效：

- 从 Godot 粒子和简单纹理开始。
- 保持战斗特效可读。
- 避免隐藏角色位置的特效。

## 许可规则

首选许可：

- CC0 / 公共领域
- 代码插件使用 MIT / BSD / Apache
- 明确允许商业使用的付费许可

谨慎使用：

- CC-BY：需要署名
- CC-BY-SA：相同方式共享要求可能影响分发
- GPL 或 LGPL 代码插件：需要审查引擎集成和分发条款

避免：

- 无许可
- 许可不明确
- 盗取的商业游戏资源
- 来源不明的转载资源

## 必需追踪

每个非原创资源都应记录在：

- [asset_register.md](asset_register.md)

必需字段：

- 资源名称
- 来源URL
- 作者
- 许可
- 商业使用状态
- 修改许可
- 署名要求
- 本地路径
- 用途
- 备注

AI生成的资源应额外记录：

- 工具/模型
- 提示词或提示词摘要
- 生成日期
- 是否经过手动编辑

## 替换策略

原型资源应易于替换。

推荐规则：

- 游戏逻辑不应依赖特定的美术文件名。
- 场景应使用清晰的占位符名称。
- 资源文件应按角色组织，而非按临时来源组织。
- 最终美术替换不应需要重写游戏脚本。
